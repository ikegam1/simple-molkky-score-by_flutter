# タスク完了時のチェックリスト

## 必須手順

1. **静的解析を実行**
   ```bash
   flutter analyze
   ```
   警告・エラーが0であることを確認

2. **フォーマット適用**
   ```bash
   dart format lib/ test/
   ```

3. **テストを実行**
   ```bash
   flutter test
   ```
   全テストがパスすることを確認

4. **Codexレビュー**（CLAUDE.md 指示より）
   - 作業開始時: `/ask-codex:review` で作業計画をレビュー
   - 作業完了時: `/ask-codex:review` で不備・不足をチェック

## 推奨手順

5. **影響範囲の確認**
   - 変更したロジックに関連するテストファイルを確認
   - スコア計算変更時は `test/game_logic_test.dart` を必ず確認

6. **コミットメッセージ**
   - 日本語で記述（既存コミット履歴に倣う）
   - 例: `feat: セルフ5ターンUI改善・4先/5先追加・v1.9.3`

## 注意点
- UIの変更はWidgetテストも更新すること
- Firebase関連の変更はオフライン動作も考慮
- 音声入力機能は実機でしか動作しない（テスト時は注意）
